### 万字文档+大学生一体化服务平台+Vue2+SpringBoot+sql文件+软件环境安装包
## 项目采用技术：
架构: B/S、MVC
系统环境：Windows/Mac
开发环境：IDEA、JDK1.8、Maven、MySQL、VSCode
技术栈：Java、MySQL、SpringBoot、Mybatis/MybatisPlus、Vue2、ElementUI、JavaScript、CSS、node.js14、Laiyui.js、HTML5（H5）

系统功能模块：前台使用HTML+Laiyui，后台使用vue+ElementUI
本系统包含管理员、用户、商家、司机4个角色。
1. 管理员
管理员在系统中拥有最高权限，负责整个系统的维护和管理。具体功能模块包括：
司机管理：管理员可以管理司机信息，包括查看、修改和删除司机信息。
商家管理：管理员可以管理商家信息，包括查看、修改和新增商家信息。
用户管理：管理员可以管理用户信息，包括查看、修改和删除用户信息。
2. 司机
司机角色主要负责拼车服务的管理。具体功能模块包括：
组队拼车管理：司机可以录入和管理组队拼车信息，包括出发地点、单人费用、座位数、到达地点和联系电话等。
拼车信息管理：司机可以查看和管理拼车订单信息，包括用户姓名、手机号以及是否支付拼车订单。
3. 商家
商家角色主要负责兼职信息的发布和管理。具体功能模块包括：
兼职信息管理：商家可以发布和管理兼职信息，包括工作地点、工作时间、薪资待遇和岗位名称等。
兼职应聘管理：商家可以下载用户的应聘简历，审核用户的应聘信息。
4. 用户
用户角色主要负责个人服务信息的管理。具体功能模块包括：
服务信息：用户可以查看和接单服务信息，包括服务详细内容。
组队拼车：用户可以查看组队拼车信息并提交拼车请求。
兼职信息：用户可以查看兼职岗位信息并投递个人简历。
闲置交易管理：用户可以发布、更改和删除闲置物品交易信息。
拼车信息管理：用户可以对提交的拼车订单进行支付和管理。
